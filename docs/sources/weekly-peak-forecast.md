# 주간 피크 전망 자료 갱신

## 출처

한국전력거래소 중앙전력관제센터 수요예측팀이 매주 금요일 발표하는 '주간 전력수요 실적 및 전망' PDF.

통상 금요일 오후에 갱신.

자료에는 다음 9일치(토~다음주 일)의 일별 예측 최대부하(GW)가 포함.

## 갱신 시점

24일 일정 중 두 번 갱신:

- D17 (5/15 금) - 그 날 발표 자료로 교체. 5/16~5/24 9일치 커버.
- D24 (5/24 일) - 직전 금요일(5/22) 발표 자료로 교체. 5/23~5/31 9일치 커버 → 시연일(5/25) 포함.

## 갱신 절차

1. PDF의 "최대부하 전망" 표에서 9일치 값을 받아 적기.
2. `lib/static-data/weekly-peak-forecast.ts`의 `LATEST_WEEKLY_FORECAST` 상수를 수정:
   - `publishedAt`을 발표일로 갱신
   - `days` 배열의 9개 항목 모두 새 값으로 교체
3. 로컬에서 메인 화면 접속 → 탄소 시계 카드 하단의 "예상 최대부하 XX.XGW · 평온/분주/긴장" 표시가 갱신되었는지 확인.
4. 커밋 메시지: `chore(data): 주간 피크 전망 자료 갱신 (YYYY-MM-DD 발표분)`.

## PDF 보관

원본 PDF는 `docs/sources/weekly-peak/` 아래 발표일을 파일명에 포함해 보관.
예: `docs/sources/weekly-peak/2026-05-08.pdf`

자료 출처를 추적할 수 있게 git에 같이 포함 (저작권상 비공개 정책이 있다면 .gitignore로 제외).

## 자료가 시연 시점을 커버하지 못할 때

`getTodayForecast`는 자료에 오늘 날짜가 없으면 `null`을 반환.
클라이언트는 이 경우 forecast 한 줄을 표시하지 않음 (CarbonClock에서 `if (todayForecast)` 가드).

깨지진 않지만 시연에서 정보가 빠져 보이니, 시연 직전 D24에 반드시 갱신 바람.
